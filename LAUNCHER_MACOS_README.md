# 🍏 HealthAI Launcher для macOS

## Описание
Скрипт `launch.sh` — универсальный лаунчер для запуска приложения HealthAI на macOS. Автоматически проверяет окружение, устанавливает зависимости, загружает модели ИИ и запускает приложение.

## 🚀 Быстрый старт

```bash
# Сделать скрипт исполняемым
chmod +x launch.sh

# Запустить приложение
./launch.sh
```

## 📋 Что делает скрипт

### 1. Проверка системы
- ✅ Проверяет наличие Python 3.10+
- ✅ Определяет версию Python
- ✅ Блокирует запуск при несоответствии версии

### 2. Виртуальная среда
- ✅ Создаёт `venv/` если отсутствует
- ✅ Активирует виртуальную среду
- ✅ Использует изолированное окружение

### 3. Зависимости
- ✅ Обновляет pip до последней версии
- ✅ Устанавливает пакеты из `requirements.txt`
- ✅ Обрабатывает ошибки установки

### 4. Модели ИИ
- ✅ Создаёт директории `ai_engine/models/` и `ml/weights/`
- ✅ Проверяет наличие моделей
- ✅ Скачивает модели по URL (если настроено)
- ✅ Создаёт демо-модели если загрузка недоступна

### 5. Целостность проекта
- ✅ Проверяет наличие критических файлов
- ✅ `main.py`, `ui/main_window.py`, `database/db_manager.py`
- ✅ Блокирует запуск при отсутствии файлов

### 6. Запуск приложения
- ✅ Запускает `main.py` через Python из venv
- ✅ Передаёт код возврата
- ✅ Корректно завершает работу

## 🎨 Цветовой вывод

Скрипт использует цветовую схему для удобства:
- 🔵 `[INFO]` — информационные сообщения (синий)
- 🟢 `[SUCCESS]` — успешные операции (зелёный)
- 🟡 `[WARNING]` — предупреждения (жёлтый)
- 🔴 `[ERROR]` — ошибки (красный)
- 🟣 `[STEP]` — шаги выполнения (голубой)

## ⚙️ Конфигурация

В начале скрипта можно настроить параметры:

```bash
PROJECT_NAME="HealthAI"              # Имя проекта
PYTHON_VERSION_REQUIRED="3.10"       # Минимальная версия Python
VENV_DIR="venv"                      # Папка виртуальной среды
REQUIREMENTS_FILE="requirements.txt" # Файл зависимостей
MODELS_DIR="ai_engine/models"        # Папка моделей
WEIGHTS_DIR="ml/weights"             # Папка весов
```

### Настройка URL моделей

Для загрузки реальных моделей отредактируйте массив `MODEL_URLS`:

```bash
MODEL_URLS=(
    "https://huggingface.co/your-model/pytorch_model.bin"
    "https://github.com/your-repo/releases/download/model.pt"
)
```

Затем раскомментируйте строку с `curl` в функции `download_models()`:

```bash
curl -L -o "$model_file" "URL_МОДЕЛИ"
```

## 🔧 Требования

- **Операционная система:** macOS 10.12+
- **Python:** 3.10 или выше
- **Bash:** 3.2+ (встроен в macOS)
- **Диск:** ~500 MB свободного места

## 🛠️ Установка Python на macOS

Если Python не установлен:

### Вариант 1: Homebrew (рекомендуется)
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install python@3.12
```

### Вариант 2: Официальный установщик
1. Перейдите на https://www.python.org/downloads/
2. Скачайте installer для macOS
3. Запустите `.pkg` файл

### Вариант 3: Pyenv
```bash
brew install pyenv
pyenv install 3.12.0
pyenv global 3.12.0
```

## 🐛 Решение проблем

### Ошибка: "Python3 не найден"
```bash
# Установите Python через Homebrew
brew install python@3.12
```

### Ошибка: "No space left on device"
```bash
# Очистите место на диске
df -h
# Или измените путь установки
```

### Ошибка: "Permission denied"
```bash
# Дайте права на выполнение
chmod +x launch.sh
```

### Ошибка при установке зависимостей
```bash
# Обновите pip вручную
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

## 🔄 Обновление

При обновлении проекта:

```bash
# Удалить старую venv
rm -rf venv

# Запустить заново
./launch.sh
```

Скрипт автоматически переустановит зависимости.

## 📞 Поддержка

При возникновении проблем:
1. Проверьте версию Python: `python3 --version`
2. Проверьте место на диске: `df -h`
3. Просмотрите логи запуска
4. Убедитесь в наличии интернета для загрузки зависимостей
