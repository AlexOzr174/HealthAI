# 🚀 HealthAI Launchers - Полная документация

## 📋 Обзор

Проект HealthAI включает в себя **3 кроссплатформенных лаунчера** для автоматической установки, настройки и запуска приложения:

| Лаунчер | Файл | ОС | Статус |
|---------|------|-----|--------|
| **macOS** | `launch.sh` | macOS, Linux | ✅ Готов |
| **Windows** | `launch.bat` | Windows 10/11 | ✅ Готов |
| **Linux** | `launch_linux.sh` | Linux (Ubuntu, Debian, Fedora) | ✅ Готов |
| **Python** | `launch_healthai.py` | Все ОС (универсальный) | ✅ Готов |

---

## 🔗 Ссылки на реальные модели ИИ

Все лаунчеры используют **реальные ML-модели** с автоматической загрузкой:

### 1. NLP Модель (Чат-бот нутрициолог)
- **Название:** `all-MiniLM-L6-v2` (Sentence Transformers)
- **Размер:** ~80 MB
- **Источник:** HuggingFace
- **URL:** `https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2/resolve/main/pytorch_model.bin`
- **Назначение:** Семантический анализ вопросов пользователя, понимание контекста

### 2. Vision Модель (Распознавание еды по фото)
- **Название:** `ResNet-50` (Microsoft)
- **Размер:** ~100 MB
- **Источник:** PyTorch Model Zoo
- **URL:** `https://download.pytorch.org/models/resnet50-0676ba61.pth`
- **Назначение:** Классификация продуктов питания на изображениях

### 3. Векторизатор (NLP)
- **Название:** `nlp_vectorizer.pkl`
- **Размер:** Создаётся локально
- **Назначение:** Векторное представление текста для чат-бота

### 4. Предиктивная модель (Аналитика веса)
- **Название:** `weight_predictor.pkl`
- **Размер:** Обучается локально
- **Назначение:** Прогнозирование изменения веса

## 📀 Инструкция для macOS

### Быстрый старт
```bash
# 1. Сделать скрипт исполняемым
chmod +x launch.sh

# 2. Запустить
./launch.sh
```

### Что делает скрипт:
1. ✅ Проверяет Python 3.9+
2. ✅ Создаёт виртуальную среду `.venv`
3. ✅ Активирует виртуальную среду
4. ✅ Устанавливает зависимости из `requirements.txt`
5. ✅ Создаёт папку `models_weights/`
6. ✅ **Загружает NLP модель** (~80MB) с HuggingFace
7. ✅ **Загружает Vision модель** (~100MB) с PyTorch
8. ✅ Создаёт векторизатор и предиктивную модель
9. ✅ Проверяет целостность проекта
10. ✅ Запускает `main.py`

### Оффлайн-режим
Если интернет отсутствует, скрипт автоматически создаст **демо-заглушки** вместо моделей и запустит приложение в ограниченном режиме.

## 🪟 Инструкция для Windows

### Быстрый старт
```cmd
# Дважды кликните на файл или запустите в командной строке:
launch.bat
```

### Что делает скрипт:
1. ✅ Проверяет Python 3.9+ в PATH
2. ✅ Создаёт виртуальную среду `.venv`
3. ✅ Активирует виртуальную среду (`Scripts\activate.bat`)
4. ✅ Обновляет pip
5. ✅ Устанавливает зависимости
6. ✅ Создаёт папки `models\nlp\` и `models\vision\`
7. ✅ **Загружает NLP модель** через curl или PowerShell
8. ✅ **Загружает Vision модель** через curl или PowerShell
9. ✅ Создаёт демо-заглушки при ошибке загрузки
10. ✅ Запускает `main.py`

### Требования:
- Windows 10/11 (для поддержки ANSI цветов)
- Python 3.9+ установлен и добавлен в PATH
- PowerShell или curl для загрузки моделей

## ⚙️ Настройки и конфигурация

### Изменение версий Python
В начале каждого скрипта укажите требуемую версию:
```bash
PYTHON_VERSION_REQUIRED="3.10"  # Минимальная версия
```

### Изменение URLs моделей
Отредактируйте переменные в `launch.sh`:
```bash
MODEL_NLP_URL="https://...новый_url..."
MODEL_FOOD_URL="https://...новый_url..."
```

### Пользовательские папки
```bash
VENV_DIR=".venv"           # Папка виртуальной среды
MODELS_DIR="models_weights" # Папка моделей
```

## 🛠️ Решение проблем

### ❌ "Python не найден"
**macOS/Linux:**
```bash
# Проверить версию
python3 --version

# Установить (macOS)
brew install python@3.10

# Установить (Ubuntu/Debian)
sudo apt update && sudo apt install python3.10 python3.10-venv
```

**Windows:**
1. Скачайте с [python.org](https://www.python.org/downloads/)
2. При установке отметьте **"Add Python to PATH"**

### ❌ "Не удалось загрузить модель"
- Проверьте подключение к интернету
- Попробуйте вручную скачать модель:
  ```bash
  curl -L -o models_weights/nlp_encoder.bin "https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2/resolve/main/pytorch_model.bin"
  ```
- Приложение запустится в **оффлайн-режиме** с демо-моделями

### ❌ "ModuleNotFoundError"
```bash
# Активировать виртуальную среду вручную
source .venv/bin/activate  # macOS/Linux
.venv\Scripts\activate     # Windows

# Переустановить зависимости
pip install -r requirements.txt --force-reinstall
```

### ❌ "Permission denied" (macOS/Linux)
```bash
chmod +x launch.sh
chmod +x launch_linux.sh
```
